# 商品化の次ステップ

このプロジェクトは、現在「デモ機が動く」段階に到達している。
次は、ココナラで受注して個別カスタマイズ納品できる商品型に整える。

## 現在の到達点

- 命式表示のデモアプリがある
- 節入りデータは1900-2200年分を静的保持している
- 大運、年運を表示できる
- 出生時刻なしに対応している
- PWA対応によりスマホアプリ風に利用できる
- AI鑑定文生成は入れていない

## 次に作るべきもの

### 1. 受注時ヒアリングシート

依頼者の流派、表示項目、鑑定文の有無、納品形式を確認する。
このシートを作ることで、標準プランと追加オプションの境界を明確にする。

### 2. ルールプロファイルの実装強化

現在は標準プロファイルをコード側で固定している。
今後は依頼者ごとに以下を切り替えられるようにする。

- 通変星の基準
- 出生時刻の扱い
- 時柱の表示有無
- 早子時、夜子時の扱い
- 蔵干表
- 大運起算方法
- 表示する占術項目

### 3. 納品テンプレート化

顧客ごとに以下を差し替えられる構造にする。

- アプリ名
- 色
- ロゴ
- 表示項目
- 鑑定文テンプレート
- 注意書き
- フッター文言

### 4. オプション商品化

ライトプランの標準範囲を固定し、追加料金対象を切り出す。

標準:

- 命式表示
- 大運
- 年運
- PWA対応
- スマホ対応
- 出生時刻なし対応

オプション候補:

- 独自流派ロジック
- 鑑定文表示
- PDF鑑定書
- 顧客入力フォーム
- 管理画面
- ロゴ、配色調整
- 既存鑑定書デザイン再現
- 独自チャート

## 優先順位

1. 受注時ヒアリングシートを作る
2. ライトプランの納品仕様を固定する
3. 顧客別カスタマイズ設定ファイルを作る
4. ココナラ出品文を作る
5. サンプル納品ページを1つ完成させる

## 商品上の見せ方

この商品は「四柱推命の自動鑑定AI」ではなく、占い師本人の鑑定業務を支える専用ツールとして見せる。

訴求軸:

- 自分の流派に合わせた命式ツールが持てる
- スマホからいつでも確認できる
- 大運、年運をすぐ確認できる
- 未来年の相談にも対応しやすい
- 鑑定文や鑑定書は必要に応じて追加できる
