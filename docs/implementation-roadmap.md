# 実装ロードマップ

## 方針

既存制作物のコードは新製品の土台にしない。

新規アプリとして作り直し、既存制作物は「どんな差分が発生しうるか」の参考資料として扱う。

## Phase 1: 静的MVP

目的:

ライトプランとして売れる最小Webアプリを作る。

実装範囲:

- 静的HTML/CSS/JS
- 生年月日・性別・出生時刻入力
- 命式計算エンジン
- 年柱・月柱・日柱・時柱
- 蔵干
- 通変星
- 十二運
- 天中殺または空亡
- 五行・陰陽バランス
- 大運
- 年運
- スマホ対応UI
- 設定JSON読み込み

検証:

- 複数サンプル日付で命式確認
- 節入り前後の確認
- 大運開始年齢の確認
- スマホ表示確認

## Phase 2: 顧客別設定

目的:

同じコード基盤で、顧客ごとの流派・表示方針に対応する。

実装範囲:

- `ruleProfile` 実装
- 出生時刻の有無切り替え
- 柱順切り替え
- 表示項目切り替え
- 通変星基準切り替え
- 蔵干表切り替え
- 大運起算設定

検証:

- 標準設定
- 時刻不要設定
- 日干基準設定
- 顧客独自表示設定

## Phase 3: オプション機能

目的:

ココナラで単価を上げる機能を追加する。

候補:

- 鑑定文テンプレート
- PDF鑑定書
- 独自チャート
- 行動キーワード表
- 顧客別の文言変更
- LINE・予約導線
- WordPress埋め込み

## Phase 4: 運用効率化

目的:

受注ごとのカスタマイズ工数を下げる。

候補:

- テンプレート生成CLI
- 顧客別設定ファイル生成
- サンプル確認ページ
- 納品チェックリスト
- 出品文・見積文テンプレート

## MVP実装の注意点

- 既存制作物からコードをコピーしない
- データ表は検証可能な形で分離する
- 計算関数はUIに依存させない
- ブラウザ表示前にテストできる関数構造にする
- ライトプランの標準仕様を先に固定する
- 顧客独自機能は後から差し込めるようにする
